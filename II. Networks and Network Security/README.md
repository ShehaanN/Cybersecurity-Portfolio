# Network Security Analysis

### 1.1 Analyzing Network Layer Communication

In this activity, I analyzed DNS and ICMP network traffic using data captured from a network protocol analyzer tool. By examining packet transmissions and error responses, I identified the network protocols involved in the cybersecurity incident and determined the root cause of the communication failure.

This analysis provided insights into:
- DNS communication over UDP
- ICMP error message handling
- Port 53 and DNS service availability
- Network troubleshooting using packet analysis tools
- Identifying protocol-level communication failures during cybersecurity incidents

The investigation helped demonstrate how network analysts use traffic analysis tools such as `tcpdump` to monitor, investigate, and diagnose network-related security and connectivity issues.

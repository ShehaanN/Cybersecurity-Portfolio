# Network Security Analysis

## Contents

### 1.1 Analyze Network Layer Communication

In this activity, I analyzed DNS and ICMP network traffic using data captured from a network protocol analyzer tool. By examining packet transmissions and error responses, I identified the network protocols involved in the cybersecurity incident and determined the root cause of the communication failure.


### 2.1 Analyze Network Attacks

In this activity, I investigated a network security incident that affected a company website and disrupted normal user access. By analyzing network traffic and server behavior, I identified the likely cause of the service interruption, analyzed how the attack occurred, and explained the negative impact it had on the website’s functionality and user trust. This hands-on analysis allowed me to demonstrate my ability to investigate network security issues and communicate findings effectively.

### 3.1 Apply OS Hardening Techniques

In this activity, I worked as a cybersecurity analyst for yummyrecipesforme.com to investigate a website security incident. I analyzed network traffic, identified how a brute force attack allowed unauthorized access to the admin account, and examined how malicious code redirected users to a fake website. Based on the investigation, I recommended OS hardening and security measures to help prevent similar attacks in the future.

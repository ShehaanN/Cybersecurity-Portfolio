# Security Incident Report: OS Hardening

## Part 1 : Identifyng the Network Protocol Involved in the Incident

The network protocol involved in the incident is the Hypertext transfer protocol (HTTP). Since the incident revolves around accessing the web server for www.yummyrecipesforme.com, HTTP traffic was identified as the protocol facilitating communication between the client browser and the web server.

The **tcpdump** logs confirm the usage of HTTP during the interaction with the website. Furthermore, the malicious file prompting the users to download new recipes was transported using the HTTP protocol, showcasing its role in the attack. This highlights how attackers exploited HTTP communication to deliver malicious content.

## Part 2 : Documenting the Incident

Several customers reported to the helpdesk that upon visiting www.yummyrecipesforme.com, they were prompted to download a file claiming to provide access to free recipes. After running the file, their computers began operating unusually slowly. Additionally, the website owner reported being unable to access the administrative account.

To investigate the incident, the cybersecurity team recreated the issue in a sandbox environment to avoid impacting the organization’s network. Using tcpdump, analysts captured and monitored network traffic while interacting with the website.

During the analysis, the browser first initiated a DNS request to resolve the IP address for www.yummyrecipesforme.com. After the DNS server responded with the correct IP address, the browser initiated an HTTP request to access the website. Once the webpage loaded, users were prompted to download and execute a malicious file disguised as a browser update.

After the file was executed, the browser initiated another DNS request for www.greatrecipesforme.com. The DNS server responded with a different IP address, and the browser then initiated an HTTP request to the malicious website. This confirmed that users were being redirected from the legitimate website to a fraudulent domain hosting malware.

Further investigation revealed that the attacker had modified the website’s source code by injecting malicious JavaScript that prompted visitors to download the executable file. The cybersecurity team also determined that the attacker gained unauthorized access to the web server through a brute force attack. The attacker successfully guessed the administrative account’s default password because no password hardening controls or protections against brute force attacks were implemented. After gaining access, the attacker changed the administrator password and embedded the malicious code into the website.

As a result of the attack, users who executed the malicious file experienced system performance issues, while the organization suffered unauthorized website modifications and reputational damage.

## Part 3: Recommend Remediations for Brute Force Attacks

To address the vulnerabilities exposed by the brute force attack, the following security measures are recommended:

### **Enforce Multi-Factor Authentication (MFA)**

- MFA requires users to verify their identity using both a password and an additional verification method, such as a one-time passcode (OTP). This added layer of security helps prevent unauthorized access even if passwords are compromised.

### **Disallow the Use of Default or Previously Used Passwords**

- Preventing the use of default and previously used passwords reduces the likelihood of attackers successfully guessing credentials through brute force techniques.

### **Implement Strong Password Policies**

- Require passwords to include a combination of uppercase letters, lowercase letters, numbers, and special characters. Strong passwords make brute force attacks significantly more difficult.

### **Monitor and Rate-Limit Login Attempts**

- Configure the system to monitor login attempts and temporarily lock accounts or block IP addresses after multiple failed login attempts. This reduces the effectiveness of brute force attacks.

By implementing these security measures, the organization can strengthen its defenses against brute force attacks and reduce the likelihood of similar incidents occurring in the future.
